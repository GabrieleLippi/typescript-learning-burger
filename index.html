What is TypeScript?

TypeScript is a language for application-scale JavaScript

Basically, it's a superset of JavaScript, adding optional static typing to the language, that compiles to clean JavaScript output.

Anders Hejlsberg, lead architect of C# and creator of Delphi and Turbo Pascal, has worked on the development of TypeScript.

History

  • 2012 - Microsoft made public TypeScript
  • 2014 - Microsoft release version 1.0
  • 2018 - TypeScript surpass JavaScript on NPM

Type Inference

When the type is not declared explicitly, TS can infer it

  • TS use the best commont type algorithm to find it. BCT picks the type that is compatible with all the other candidates
  • When no best commont type is found, the inference is the union array type

Type Annotations

Basic Types

types are annotated using :TypeAnnotation syntax


                                        const num: number = 123;

                                        function identity(num: number): number {
                                            return num;
                                        }

JavaScript primitive types are well represented in the TypeScript type system

We can specify that the variable is an Array using []

Interfaces are the core way in TypeScript to compose multiple type annotations into a single named annotation


                                        interface Name {
                                            first: string;
                                            second: string;
                                        }

                                        var name: Name;
                                        name = {
                                            first: 'John',
                                            second: 'Doe'
                                        };

It's also possible to do Inline Type Annotation, but it's not the best practice for readability reasons

Special Types

These types have special meaning in TS: any, null, undefined, void.

  • any basically tells the compiler to not consider the type of the variable
  • null and undefined can be assigned to any other type.
  • void is used to signify that a function does not have a return type

Generics: when you want to enforce a constraint between various variables


                                        function reverse(items: T[]): T[] {
                                            var toreturn = [];
                                            for (let i = items.length - 1; i >= 0; i--) {
                                                toreturn.push(items[i]);
                                            }
                                            return toreturn;
                                        }

                                        const sample = [1, 2, 3];
                                        const reversed = reverse(sample);
                                        console.log(reversed); // 3,2,1

                                        // Safety!
                                        reversed[0] = '1';     // Error!
                                        reversed = ['1', '2']; // Error!

                                        reversed[0] = 1;       // Okay
                                        reversed = [1, 2];     // Okay

Union Types: denoted by a |

Intersection Types


                                        function extend(first: T, second: U): T & U {
                                          let result =  {};
                                          for (let id in first) {
                                              result[id] = first[id];
                                          }
                                          for (let id in second) {
                                              if (!result.hasOwnProperty(id)) {
                                                  result[id] = second[id];
                                              }
                                          }
                                          return result;
                                      }

Namespace

Namespaces provide you with a convenient syntax around a common pattern used in JavaScript


					(function(something) {

					    something.foo = 123;

					    })(something || (something = {}))

This is commonly used in the JavaScript land for making sure that stuff doesn't leak into the global namespace: therefore TypeScript provides the namespace keyword to group these


				      namespace Utility {
					export function log(msg) {
					  console.log(msg);
					}
					export function error(msg) {
					  console.error(msg);
					}
				      }
				      // usage
				      Utility.log('Call me');
				      Utility.error('maybe!');

Avoid export default:

Auto import quickfix works better

Pro tips:

  1. Don’t use public accessor within classes: all members within class are public by default, and is not "valid/idiomatic javascript"
  2. Don’t use private accessor within Component class: private accessor won't make your properties/methods on class private during runtime and you should almost never need to work directly with React Component instance nor accessing its class properties.
  3. Don't use Enums: compiled output generates unnecessary boilerplate
  4. Always provide explicit type for children Props
  5. Use type inference for defining Component State or DefaultProps
  6. Don’t use method declaration within interface/type alias
  7. Don’t use JSX.Element to annotate function/component return type or children/props
